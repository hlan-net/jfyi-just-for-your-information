JFYI

Just For Your Information — a passive MCP server and analytics hub with Profile-Guided Optimization.

MCP FastAPI SQLite Apache-2.0

What it does

JFYI passively collects interaction traces from your agents and tools, scores friction in real time, and feeds profile-guided hints back through MCP. It ships with a dashboard, a REST API, and a Helm chart for Kubernetes.

Quickstart

# Container (multi-arch: linux/amd64, linux/arm64)
docker run -p 8080:8080 -v jfyi-data:/data ghcr.io/hlan-net/jfyi-just-for-your-information:latest

# Helm chart (OCI, Helm 3.8+)
helm install my-jfyi \
  oci://ghcr.io/hlan-net/charts/jfyi-mcp-server \
  --namespace jfyi-system --create-namespace \
  --set persistence.size=2Gi

Roadmap

The planned improvements address Context Rot — the degradation in reasoning quality as the context window fills. Each phase layers on the previous.

Phase 1 — Foundation  ·  v2.1.0

Context footprint reduction

  • Progressive Disclosure — expose a single router tool upfront; expand schemas on demand Planned
  • Payload Minification — compact JSON and TOON serialization at the prompt boundary Planned

Phase 2 — Memory Architecture  ·  v2.2.0

Session state management

  • Compiled View Memory — artifact handles instead of raw content; local script execution Planned
  • Context Compaction — rolling summaries and prefix-cache-aware prompt structure Planned

Phase 3 — Advanced Retrieval  ·  v2.3.0

Semantic tool and instruction retrieval

  • Instruction-Tool Retrieval (ITR) — dynamic per-step selection of the minimal relevant instruction and tool subset Planned
